KEY FEATURES
================================================================================

✅ Multi-Source Data Processing
   - CSV (structured)
   - GitHub (unstructured)
   - LinkedIn (placeholder ready)

✅ Smart Merging
   - Source priority strategy (CSV > GitHub > LinkedIn)
   - Confidence-based selection
   - Most-complete strategy
   - Array deduplication

✅ Field Normalization
   - Phone: E.164 format (+1234567890)
   - Date: YYYY-MM format (2023-01)
   - Country: ISO 3166-1 alpha-2 (US, GB, CA)
   - Skills: Canonical names (JavaScript, React, Python, etc.)

✅ Confidence Scoring
   - 0.0 (unreliable) to 1.0 (highly confident)
   - Based on: presence, agreement, well-formedness, source diversity

✅ Data Provenance Tracking
   - Know where each field came from (CSV, GitHub, merged)
   - Know how it was processed (selection, normalization, merge)
   - Full traceability for audit purposes

✅ Configurable Runtime Behavior
   - Select which fields to include
   - Rename fields dynamically
   - Set normalization rules per field
   - Configure missing value handling (null, omit, error)
   - Choose merge strategy

   → Cannot merge unstructured sources, process CSV only

2. ✅ Conflicting Company Names
   → Priority-based resolution with conflict logging

3. ✅ Malformed Phone Numbers
   → Sanitize or return null (never invent data)

   → Parse flexibly (MM/DD/YYYY, Jan 15 2023, 2023-01-15, 2023, etc.)

   → Handle per configuration (null, omit, or error)

   → Case-insensitive merging of emails, phones, skills
